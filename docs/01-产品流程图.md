# 「最佳干饭人」产品流程图

> 个人主体版 MVP · 两人配对 · 无AI · 无社交广场

---

## 2. 主流程 A：首次登录

```
[打开小程序]
     │
     ▼
[检查 wx.getStorageSync('openid') 是否存在？]
     │
  ┌──┴──┐
  │不存在│
  └──┬──┘
     ▼
[登录页]
  - 展示 logo + 应用名 + 启动语（梦幻治愈风）
  - 主按钮「微信一键登录」
     │
     ▼
[用户点击登录]
     │
     ▼
[wx.getStorageSync/Profile → 获取昵称头像授权]
     │
  ┌──┴──┐
  │拒绝 │
  └──┬──┘
     ▼
[仍用匿名 openid 进入首页，弹窗提示"未授权将影响搭子功能"]
     │
     ▼
[wx.login() → code → 云函数 login]
     │
     ▼
[云函数用 code 调 wx-server-sdk 获取 openid]
     │
     ▼
[存入 users 集合（首次注册）]
     │
     ▼
[本地缓存 openid + 跳转到首页]
```

## 3. 主流程 B：拍照打卡（核心）

```
[首页]
  - 顶部：今日日期 + 用户头像昵称
  - 中部：「今日打卡」空状态 OR 时间线列表
  - 底部：右下角悬浮按钮【开干！】
  - 顶部右上角：「配对」图标入口
     │
     │ 点 【开干！】
     ▼
[拍照打卡页 /pages/camera/camera]
  - 顶部：「记录这一餐」
  - 区域A：照片预览（默认占位图）
  - 区域B：餐次选择（早/午/晚/加餐）— 横向胶囊按钮
  - 区域C：标签多选（米饭/面食/蔬菜/汤/甜品/水果/速食/饮料）
  - 区域D：备注输入框（最多 50 字）
  - 底部主按钮：【晒一晒】（亮色马卡龙）
     │
     │ 点 【晒一晒】
     ▼
[前端校验]
  ├─ 照片已选？否 → 提示「先来一张照片吧～」
  ├─ 餐次已选？否 → 提示「选个餐次吧～」
  └─ 全部通过
     │
     ▼
[云函数 createRecord 入参]
  { imageUrl, mealType, tags[], note }
     │
     ▼
[云函数流程]
  1. imgSecCheck 同步校验图片（不通过则返回 fail）
  2. 内容安全校验备注文本（不通过则截断或拒绝）
  3. 写入 meals 集合
  4. 返回 { success, recordId }
     │
     ▼
[前端：toast "打卡成功 ✨" → 返回首页]
  首页时间线新增一条
```

### 异常分支

| 场景 | 处理 |
|------|------|
| 用户拒绝相机权限 | 引导开启设置 |
| 图片加载失败 | 提示重试 |
| 内容安全不通过 | 提示"图片不符合规范"（不暴露具体原因） |
| 云函数超时 | 提示"网络小情绪，重试一下" |
| 当日已打卡相同餐次 | 提示"这餐已经记过啦"，但仍允许保存为新一条 |

## 4. 主流程 C：配对干饭（核心差异化）

### 4.1 发起配对（用户A）

```
[首页] → 点右上角 [配对] 图标
     │
     ▼
[配对页 /pages/pair/pair]
  - 顶部 Tab：发起配对 | 输入配对码
  - 当前页（发起）：
    · 大标题「邀请你的搭子一起干饭」
    · 副文案「配对后双方可看到彼此的打卡」
    · 主按钮「生成配对码」（大尺寸柔焦）
     │
     ▼
[点击生成]
     │
     ▼
[云函数 pairCreate]
  - 生成 6 位大写字母+数字（如 K7F2X9）
  - 写入 pairCodes 集合 { code, ownerOpenid, status:'pending', expireAt: now+24h }
  - 记录当前用户 partnerOpenid 临时置空
     │
     ▼
[前端展示配对码]
  - 大字号显示配对码（带颜色装饰）
  - 副文案「码的有效期 24 小时」
  - 三个操作按钮：
    · 复制配对码
    · 分享给朋友（onShareAppMessage）
    · 我已发送，等待对方加入
  - 轮询 / 实时：3秒一次查 status
     │
     ▼
[对方输入后，status 变为 confirmed]
     │
     ▼
[前端展示成功态]
  - 「配对成功！搭子：[B的昵称]」
  - 显示 B 的头像
  - 主按钮「去打个卡庆祝一下」
  - 次按钮「看看搭子的记录」
```

### 4.2 加入配对（用户B）

```
[首页] → 点右上角 [配对] 图标
     │
     ▼
[配对页] → 切到 [输入配对码] Tab
  - 输入框（6位，自动转大写）
  - 主按钮「加入配对」
     │
     ▼
[点击加入]
  前端校验：6位且仅含字母数字
     │
     ▼
[云函数 pairJoin]
  1. 校验码存在 + 未过期
  2. 校验当前用户无 partnerOpenid
  3. 校验发起方 ownerOpenid 也无 partnerOpenid
  4. 双向写入 partnerOpenid
  5. 更新 pairCodes.status = 'confirmed'
  6. 返回 { success, partner: {openid, nickName, avatarUrl} }
     │
     ▼
[前端展示成功态]（同 4.1 末尾）
```

### 4.3 异常分支

| 场景 | 处理 |
|------|------|
| 码不存在 | 「配对码不对，再确认下？」 |
| 码过期 | 「配对码已过期，请让对方重新生成」 |
| 当前用户已有搭子 | 「你已经有搭子啦，要先解除吗？」→ 弹窗确认 → 解绑后重试 |
| 发起方已有搭子 | 「对方已经有搭子了，等 TA 处理一下」 |
| 自己配对自己 | 「不能和自己配对哦」 |

## 5. 主流程 D：搭子查看

```
[首页] → 顶部「我的搭子」入口（无配对时显示引导）
     │
     ▼
[搭子页 /pages/partner/partner]
  - 顶部：搭子信息卡
    · 头像 + 昵称
    · 配对天数（X 天）
    · 解除配对按钮（红色描边）
  - 中部：搭子今日打卡
    · 有打卡：列出今日所有餐次卡片
    · 无打卡：「TA 还没开干，去提醒下？」
  - 底部：搭子最近 7 天打卡日历（7个圆点，已打卡亮色）
  - 操作：「给搭子点个赞」按钮（送一颗小心心，仅 1 次/天）
     │
     ▼
[点打卡卡片] → 跳 [打卡详情 /pages/record-detail/record-detail]
  - 顶部大图
  - 餐次 + 时间
  - 标签
  - 备注
```

## 6. 主流程 E：历史记录

```
[首页] → 顶栏 [我的] 头像 → [个人中心 /pages/profile/profile]
     │
     ▼
[个人中心]
  - 用户信息卡
  - 数据统计：累计打卡 X 餐 / 已配对 X 天
  - 入口列表：
    · 我的历史 → 历史页
    · 我的搭子 → 搭子页
    · 关于我们 → 关于页
    · 解除配对（仅配对时显示）
     │
     ▼
[点 我的历史]
     │
     ▼
[历史页 /pages/history/history]
  - 顶部：月份选择器
  - 主体：按日期分组的时间线
    · 某日卡片（日期 + 当日餐次数）
    · 展开：当日所有餐次缩略卡
  - 底部：上拉加载更多
```

## 7. 全局异常流程

### 7.1 网络异常
```
[任意页面发起请求]
     │
     ▼
[云函数超时/失败]
     │
     ▼
[全局拦截器]
  - 弹 toast「网络小情绪，再试一下～」
  - 保留当前页面状态
  - 关键操作（打卡）可手动重试
```

### 7.2 登录态失效
```
[任意请求返回 code: 'AUTH_EXPIRED']
     │
     ▼
[清除本地 openid]
     │
     ▼
[跳转到登录页]
```

### 7.3 内容安全拦截
```
[打卡/分享触发 imgSecCheck/textSecCheck]
     │
     ▼
[不通过]
     │
     ▼
[前端提示「内容不太合适，换一张图试试」]
[云函数不写入数据库]
```
