# 「最佳干饭人」云函数接口设计文档

> 全部接口通过 **微信云开发云函数** 调用，请求方法 `wx.cloud.callFunction({ name, data })`
> 统一返回结构 `{ success: boolean, data?: any, code?: string, message?: string }`

---

## 0. 通用约定

### 0.1 错误码

| code | 含义 | 客户端处理 |
|------|------|-----------|
| `OK` | 成功 | - |
| `PARAM_INVALID` | 参数错误 | 提示重试 |
| `AUTH_REQUIRED` | 未登录 | 跳登录页 |
| `AUTH_EXPIRED` | 登录态失效 | 清本地态并跳登录页 |
| `IMG_CHECK_FAIL` | 图片内容安全不通过 | 「换张图试试」 |
| `TEXT_CHECK_FAIL` | 文本内容安全不通过 | 「换个说法试试」 |
| `CODE_NOT_FOUND` | 配对码不存在 | 「码不对哦」 |
| `CODE_EXPIRED` | 配对码已过期 | 「码已过期」 |
| `ALREADY_PAIRED` | 已有配对 | 弹窗提示 |
| `SELF_PAIR` | 自己配对自己 | 弹窗提示 |
| `NETWORK_ERROR` | 网络错误 | toast 提示重试 |
| `INTERNAL_ERROR` | 内部错误 | toast 提示 |

### 0.2 通用入参

所有需要鉴权的接口，前端传入 `cloud.getWXContext().OPENID`（云函数端自动获取），
无需前端传 openid。

## 1. login

**用途**：微信登录，换取 openid 并初始化用户记录

### 入参
```js
{
  nickName?: string,    // 授权后的昵称
  avatarUrl?: string    // 授权后的头像
}
```

### 出参
```js
{
  success: true,
  data: {
    openid: 'oXyz...',
    isNew: true,        // 是否新用户
    user: {
      nickName, avatarUrl, partnerOpenid: '', createdAt
    }
  }
}
```

### 流程
1. 调 `cloud.getWXContext()` 获取 openid
2. 查询 `users` 集合，无则插入
3. 返回用户信息

## 2. createRecord

**用途**：创建一条打卡记录（含内容安全审核）

### 入参
```js
{
  imageUrl: string,        // 微信云存储 fileID 或临时url
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'snack',
  tags: string[],          // ['米饭', '蔬菜', ...] 最多 5 个
  note: string             // 最多 50 字
}
```

### 出参
```js
{
  success: true,
  data: {
    recordId: 'xxx',
    createdAt: 1700000000000
  }
}
```

### 流程
1. 鉴权（取 openid）
2. **imgSecCheck 同步校验** imageUrl
   - 不通过 → `IMG_CHECK_FAIL`
3. **textSecCheck 同步校验** note
   - 不通过 → `TEXT_CHECK_FAIL`
4. 写入 `meals` 集合
5. 返回 recordId

### 数据落库
```js
{
  _openid: 'oXyz...',
  date: '2024-11-15',     // YYYY-MM-DD（用户时区）
  time: '12:30',          // HH:mm
  timestamp: 1700000000000,
  imageUrl: 'cloud://...',
  mealType: 'lunch',
  tags: ['米饭', '蔬菜'],
  note: '今天吃得很开心',
  createdAt: Date.now()
}
```

## 5. pairCreate

**用途**：发起配对，生成 6 位配对码

### 入参
无

### 出参
```js
{
  success: true,
  data: {
    code: 'K7F2X9',
    expireAt: 1700086400000,  // 24h 后
    expireIn: 86400           // 秒
  }
}
```

### 流程
1. 鉴权
2. 校验当前用户 `partnerOpenid === ''`（否则 `ALREADY_PAIRED`）
3. 查 `pairCodes`：若有未过期 pending 码则复用
4. 否则生成新码（6位大写字母+数字，去掉易混字符 0O1I）
5. 写入 `pairCodes` 集合
6. 返回码

### 配对码生成规则
```js
const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'; // 去掉 0/1/O/I
let code = '';
for (let i = 0; i < 6; i++) {
  code += chars[Math.floor(Math.random() * chars.length)];
}
```

## 6. pairJoin

**用途**：输入配对码加入

### 入参
```js
{
  code: 'K7F2X9'
}
```

### 出参
```js
{
  success: true,
  data: {
    partner: {
      openid: 'oXyz...',
      nickName: '小明',
      avatarUrl: 'cloud://...',
      pairedAt: 1700000000000
    }
  }
}
```

### 流程
1. 鉴权
2. 校验码格式（6位）
3. 查 `pairCodes`：code 存在？
4. 校验未过期（`expireAt > now`），否则 `CODE_EXPIRED`
5. 校验 status === 'pending'，否则提示
6. 校验 `ownerOpenid !== currentOpenid`，否则 `SELF_PAIR`
7. 校验发起方 `users.partnerOpenid === ''` 且 当前用户 `partnerOpenid === ''`
   - 否则 `ALREADY_PAIRED`
8. **事务**：
   - 更新 `users[owner].partnerOpenid = currentOpenid`
   - 更新 `users[current].partnerOpenid = ownerOpenid`
   - 更新 `pairCodes.status = 'confirmed'`
9. 返回对方信息

---

## 7. pairUnbind

**用途**：解除配对

### 入参
无

### 出参
```js
{ success: true }
```

### 流程
1. 鉴权
2. 查 `users[current]`
3. 校验 `partnerOpenid !== ''`
4. 事务：
   - 当前用户 `partnerOpenid = ''`
   - 对方 `partnerOpenid = ''`
5. 不删除历史打卡

## 8. getPartner

**用途**：获取搭子的今日 + 最近7天打卡

### 入参
无

### 出参
```js
{
  success: true,
  data: {
    partner: {
      openid, nickName, avatarUrl,
      pairedAt
    },
    pairedDays: 12,           // 配对天数
    today: {
      date: '2024-11-15',
      records: [Record]
    },
    weekStats: [
      { date: '2024-11-09', count: 3 },
      { date: '2024-11-10', count: 2 },
      ...
    ],
    likeSentToday: false      // 今天是否已点赞
  }
}
```

### 流程
1. 鉴权
2. 查 `users[current].partnerOpenid`，空则返回 `ALREADY_PAIRED`
3. 并行查：
   - 对方用户信息
   - 对方今日 meals
   - 对方最近 7 天 meals（按日期聚合）
4. 计算配对天数 = (now - pairedAt) / 86400000
5. 查 `likes` 集合判断今天是否已点赞

## 11. 数据库集合设计

### 11.1 users

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 系统 |
| _openid | string | 微信 openid（系统） |
| nickName | string | 昵称 |
| avatarUrl | string | 头像 URL |
| partnerOpenid | string | 搭子的 openid，空字符串表示无 |
| pairedAt | number | 配对时间戳 |
| createdAt | number | 注册时间戳 |

**索引**：`_openid`（系统自带）

### 11.2 meals

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 系统 |
| _openid | string | 系统 |
| date | string | YYYY-MM-DD |
| time | string | HH:mm |
| timestamp | number | 完整时间戳 |
| imageUrl | string | 云存储 fileID |
| mealType | string | breakfast/lunch/dinner/snack |
| tags | string[] | 多选标签 |
| note | string | 备注（≤50字） |
| createdAt | number | - |

**索引**：
- `_openid + date` 复合索引（首页 + 历史用）

### 11.3 pairCodes

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 系统 |
| code | string | 6位配对码（唯一） |
| ownerOpenid | string | 发起人 |
| status | string | pending/confirmed/expired |
| expireAt | number | 过期时间戳 |
| createdAt | number | - |

**索引**：
- `code` 唯一索引
- `expireAt`（用于定时清理）

### 11.4 likes（二期）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 系统 |
| _openid | string | 点赞人 |
| partnerOpenid | string | 被点赞人 |
| date | string | 点赞日期 |
| createdAt | number | - |

**索引**：
- `_openid + date` 唯一（保证每天1次）
