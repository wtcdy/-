# 最佳干饭人 · PRD v3.1

> 个人项目 · 朋友间 1:1 配对干饭工具 · 梦幻治愈风格
> **v3.1 变更**：拍照页三类标签全部"必选"（餐次/正餐速食/食物类别）；历史页去除旧细粒度食物标签，**仅保留 5 大类别 + 按时/延时**
> 本版本累计：打卡阈值、正餐/速食二选一、5 大食物类别（多选）、周统计（仅工作日）

---

## 1. 应用概述

**应用名称**：最佳干饭人

**应用描述**：趣味饮食打卡微信小程序，用于朋友亲人之间 1:1 配对监督彼此是否按时健康饮食。通过饭前拍照上传、手动选餐次与饮食分类、打卡阈值与延期标识、工作日周统计，帮助用户养成规律吃饭的好习惯。

**美学风格**：梦幻治愈风格，低对比度、轻盈柔焦质感、马卡龙色搭配、圆角柔边元素为主。

**主体与定位**：个人主体非盈利项目，仅供朋友间私密使用，无社交广场、无公开分享。

---

## 2. 用户与使用场景

**目标用户**：希望与 1 位朋友/亲人互相监督吃饭的朋友群体。

**核心使用场景**：
- 饭前拍照记录饮食内容（早/午/晚/加餐）
- 判断是否按时打卡：按时（绿色 ✓） / 延期（黄色 ⏰） / 缺卡（灰色 —）
- 区分正餐与速食（每次打卡二选一）
- 周末休息，工作日累计统计周报
- 朋友间 1:1 配对，互相查看对方打卡

## 4. 核心功能模块

### 4.1 登录

- 微信一键授权登录
- 缓存 openid + 头像 + 昵称
- 个人主体，类目：**工具 - 健康管理**

### 4.2 拍照打卡

#### 4.2.1 餐次（4 选 1）

| 餐次 | 截止时间 | 延期截止 | 备注 |
|------|----------|----------|------|
| 早餐 | 10:00 | 11:00 | 超过 11:00 视为缺卡 |
| 午餐 | 14:00 | 15:00 | 超过 15:00 视为缺卡 |
| 晚餐 | 21:00 | 22:00 | 超过 22:00 视为缺卡 |
| 加餐 | 无限制 | — | 任何时间都可 |

#### 4.2.2 打卡状态

| 状态 | 条件 | 视觉标识 | 颜色 |
|------|------|----------|------|
| **按时** | 当前时间 ≤ 截止时间 | 🟢 ✓ | 薄荷绿 |
| **延期** | 截止时间 < 当前时间 ≤ 延期截止 | 🟡 ⏰ | 蜜桃橙 |
| **缺卡** | 超过延期截止（按餐次计算） | ⚪ — | 雾霾蓝（仅在总结时出现） |

> 加餐无阈值概念，始终为"按时"。

#### 4.2.3 分类标签（必选两类）

每次打卡必须选择两类标签：

**A. 正餐 / 速食（单选，必选）**

| 标签 | 说明 | 视觉 |
|------|------|------|
| 正餐 🍱 | 正常家常饭、外卖便当、餐厅点餐 | 主色（粉） |
| 速食 🍔 | 泡面、快餐、汉堡、便利店便当等 | 蜜桃橙 |

**B. 食物类别（多选，至少 1 个，最多 5 个）**【v3 升级：从 10 个细粒度 → 5 大类别】

| 类别 | 涵盖范围 | emoji | 视觉色 |
|------|---------|-------|--------|
| 主食 | 米饭、面食、面包、馒头、饺子、意面等 | 🍚 | 奶油黄 `#FFE7A0` |
| 蔬菜水果 | 各种蔬菜、水果 | 🥗 | 薄荷绿 `#C6EBC9` |
| 肉蛋水产 | 猪肉、牛肉、鸡肉、鱼、虾、蛋、奶制品 | 🍗 | 蜜桃橙 `#FFC7A0` |
| 汤粥饮品 | 汤、粥、豆浆、咖啡、奶茶、果汁 | 🍵 | 雾霾蓝 `#C7E0F0` |
| 甜品零食 | 蛋糕、巧克力、饼干、薯片、冰淇淋等 | 🍰 | 薰衣紫 `#D7C7F0` |

> **设计依据**：将 10 个细粒度食物合并为 5 大类别，降低打卡心智负担；同时为每个类别分配独立色相，让"今日饮食结构"一目了然。颜色全部来自既有马卡龙色板，与"梦幻治愈"调性一致。

#### 4.2.4 备注

- 最多 50 字，可选
- 走内容安全审核

#### 4.2.5 拍照

- 调起相机或相册
- 客户端压缩到 ≤ 1MB
- 上传云存储
- 强制走 `imgSecCheck` 内容安全

### 4.3 首页（今日打卡）

展示今日所有餐次（早/午/晚），按时间顺序：

- **未到时间**：餐次卡片为灰色，显示"待开干"
- **按时已打卡**：薄荷绿边框 + 🟢 ✓
- **延期已打卡**：蜜桃橙边框 + 🟡 ⏰
- **缺卡**（超过延期截止）：雾霾蓝边框 + ⚪ —

**进度环**（顶部）展示今日 3 餐状态（按时/延期/缺卡/待打卡）。

**FAB**：右下角"开干"，点击进入拍照打卡页。

### 4.4 配对

- 1:1 私密数据共享，**不支持多人**
- 6 位配对码（字母+数字），24 小时过期
- 双向确认机制：发起方生成码、对方输入码、双方都解绑旧关系后才能新配对
- 解除配对：弹窗确认，历史记录保留

### 4.5 搭子

- 搭子信息卡：头像、昵称、配对天数、解除按钮
- 搭子今日打卡（按时/延期/缺卡都用对应颜色标识）
- 搭子本周 5 个工作日统计
- 给搭子点赞（每天 1 次）

### 4.6 周总结（新增 · 核心）

#### 4.6.1 时间维度

- **统计范围**：本周一 00:00 ~ 本周日 23:59
- **有效打卡日**：仅 **工作日**（周一 ~ 周五）纳入统计
- **周末规则**：周六、周日的打卡**不计入**统计（视为休息日）
- **节假日**：本期不识别法定节假日，简单按周一周五统计

#### 4.6.2 统计指标

| 指标 | 计算方式 | 展示形式 |
|------|----------|----------|
| 总打卡数 | 工作日内的所有打卡条数 | 大号数字 |
| 正餐数 | category=正餐 的条数 | 数字 + 占比 |
| 速食数 | category=速食 的条数 | 数字 + 占比 |
| 按时率 | 按时条数 / 总条数 | 百分比 + 进度环 |
| 延期率 | 延期条数 / 总条数 | 百分比 |
| 缺卡数 | 工作日预期 3 餐 × 5 天 - 实际打卡 | 数字 |
| 标签 Top3 | 出现次数最多的 3 个食物标签 | 横向胶囊 |
| 日历视图 | 5 个工作日圆点按时/延期/缺卡着色 | 圆点日历 |

#### 4.6.3 入口

- 首页顶部"本周总结"卡片（周一到周日滚动展示）
- "我的"页"本周总结"入口
- 周日晚上主动推送（如已订阅消息）

### 4.7 历史

- 按月份切换
- 按日期分组，每日卡片显示餐次数 + 按时/延期标识
- 支持查看任意打卡详情

### 4.8 我的

- 用户信息
- 累计数据：总打卡数、配对天数
- 入口：本周总结、历史、搭子、解除配对、关于

## 5. 业务规则

### 5.1 打卡阈值规则（核心）

```
当前时间 T
餐次截止时间 D
餐次延期截止时间 D'

if T ≤ D:  status = 'on_time'   🟢
elif D < T ≤ D':  status = 'late'  🟡
else:  status = 'missed' ⚪
```

服务器时间为准（避免用户改手机时间作弊）。

### 5.2 加餐规则

- 加餐无时间限制
- 加餐计入"总打卡数"
- 加餐**不计入**正餐/速食分布
- 加餐**不参与**按时率/延期率（视为零干扰项）

### 5.3 配对规则

- 1 个用户最多 1 个搭子
- 解除配对不影响历史数据
- 配对码 24 小时过期，最多同时存在 1 个 pending 码
- 不能配对自己

### 5.4 内容安全

- 每次图片上传走 `imgSecCheck`
- 每次备注文本走 `textSecCheck`（限 50 字）
- 不通过则拒绝写入，提示用户

### 5.5 工作日统计

- 周一 ~ 周五：纳入统计
- 周六、周日：纳入展示但**不纳入数字**
- 周日 23:59 自动生成"本周总结"
- 周一 00:00 切换到"上周总结"并保留

## 6. 数据模型

### 6.1 meals 集合（更新）

```js
{
  _id, _openid,
  date: '2024-11-15',          // YYYY-MM-DD
  time: '08:15',                // HH:mm
  timestamp: 1700000000000,     // 用于判断状态
  imageUrl: 'cloud://...',
  mealType: 'breakfast',        // breakfast/lunch/dinner/snack
  mainCategory: 'meal',         // 【新增】meal=正餐 / fast=速食
  tags: ['米饭', '蔬菜'],       // 食物类型
  note: '今天吃得很开心',
  status: 'on_time',            // 【新增】on_time / late
  createdAt: 1700000000000
}
```

**索引**：`_openid + date` 复合索引

### 6.2 weeklySummary 集合（新增）

```js
{
  _id, _openid,
  weekStart: '2024-11-11',      // 周一日期
  weekEnd: '2024-11-15',        // 周五日期
  totalMeals: 12,               // 总打卡数
  mealCount: 9,                 // 正餐数
  fastCount: 3,                 // 速食数
  mealRate: 75,                 // 正餐占比 %
  onTimeCount: 10,              // 按时数
  lateCount: 2,                 // 延期数
  onTimeRate: 83,               // 按时率 %
  missedCount: 3,               // 缺卡数
  topTags: [                    // 出现最多的 3 个
    { name: '蔬菜', count: 8, rate: 67 },
    { name: '米饭', count: 7, rate: 58 },
    { name: '肉类', count: 5, rate: 42 }
  ],
  dailyStats: [                 // 5 个工作日
    { date: '2024-11-11', onTime: 2, late: 1, missed: 0 },
    ...
  ],
  generatedAt: 1700000000000
}
```

### 6.3 pairCodes（不变）

不变。

### 6.4 users（更新）

新增字段：

```js
{
  ...

  likedPartnerDates: ['2024-11-15']  // 已点赞搭子的日期列表（用于限制每天 1 次）
}
```

## 7. 接口更新

### 7.1 createRecord 更新

**入参新增字段**：
```js
{
  imageUrl, mealType, tags, note,
  mainCategory: 'meal' | 'fast'   // 【新增】
}
```

**新增逻辑**：
- 服务器端根据 `timestamp` 和 `mealType` 计算 `status`
- 加餐的 `status` 固定为 `on_time`

### 7.2 getWeeklySummary（新增）

**入参**：
```js
{ weekStart: '2024-11-11' }   // 可选，默认本周
```

**出参**：
```js
{
  success: true,
  data: {
    weekStart, weekEnd,
    totalMeals, mealCount, fastCount, mealRate,
    onTimeCount, lateCount, onTimeRate, missedCount,
    topTags, dailyStats,
    weekRangeText: '11.11 - 11.15'
  }
}
```

### 7.3 getCurrentWeekStats（新增）

**入参**：无

**出参**：精简版周统计（首页卡片用）

```js
{
  success: true,
  data: {
    daysPassed: 3,           // 已过工作日数
    totalDays: 5,
    onTimeRate: 80,
    mealRate: 75,
    todayStatus: 'on_time' | 'late' | 'pending'
  }
}
```

## 9. 异常与边界

| 场景 | 处理 |
|------|------|
| 缺卡 | 总结里以灰色显示，不影响"按时率"分母 |
| 加餐 | 不参与正餐/速食分布，单独统计 |
| 周末打卡 | 计入"总打卡"但**不参与按时率**、不参与正餐速食分布 |
| 节假日 | 本期不识别，视为正常工作日 |
| 配对码过期 | 提示重新生成 |
| 同时多次配对 | 同一时间只能有 1 个 pending 码 |
| 内容安全不通过 | 拒绝保存，提示换图 |
