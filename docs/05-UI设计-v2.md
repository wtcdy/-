# UI 设计 v2 · 重设计方案

> 在 v1 基础上加入 3 个新功能：
> 1. 打卡阈值（按时/延期/缺卡，三色标识）
> 2. 正餐/速食（顶部二选一）
> 3. 周总结（工作日统计）

---

## 1. 视觉令牌扩展

### 1.1 新增状态色

| 状态 | 主色 | 浅色背景 | 含义 |
|------|------|----------|------|
| 按时 `on_time` | `#5C8C72` 薄荷绿 | `rgba(200,230,217,0.3)` | 截止时间内 |

## 9. 关键交互流

### 9.1 拍照 → 阈值反馈

```
用户打开拍照页
  → 选餐次
  → 实时计算状态
  → 选正餐/速食（必选）
  → 选食物类型
  → 写备注
  → 看到"当前 14:30，午餐 14:00 截止，已延期 30min"提示
  → 点击保存
  → 服务器再次校验状态（防作弊）
  → 写库
  → 返回首页，看到打卡卡边框为蜜桃橙 + 🟡 延期 30min
```

### 9.2 首页 → 周总结

```
用户点"本周小结卡"
  → 进入周总结页
  → 加载 getWeeklySummary
  → 显示大圆环 + 分布 + Top3 + 5 日日历
  → 点"查看历史" → 跳历史页
```

### 9.3 周末访问

```
周六打开小程序
  → 首页本周小结卡显示"周末好好休息～"
  → 周总结页顶部提示"今天是周末，工作日统计明天更新"
  → 周末打卡仍可记录，但只展示在历史中，不纳入统计
```

## 10. 组件库

### 10.1 状态徽章 `status-badge`

```
薄荷绿底: 🟢 按时
蜜桃橙底: 🟡 延期 30min
雾霾蓝底: ⚪ 缺卡
浅灰底:   ⏰ 待打卡
```

### 10.2 主分类徽章 `main-category-badge`

```
粉底: 🍱 正餐
橙底: 🍔 速食
```

### 10.3 圆环进度 `ring-progress`

- 大圆环（240rpx）：按时率
- 中圆环（160rpx）：正餐占比
- 小圆环（80rpx）：单个餐次的完成度

### 10.4 周日历 `week-calendar`

- 5 列工作日
- 每列：日期 + 圆点（按时/延期/缺卡/未来）+ 餐数 + 状态

## 11. 视觉风格一致

- 维持梦幻治愈 + 马卡龙色
- 状态色薄荷绿/蜜桃橙/雾霾蓝/浅灰 都是已有色板
- 圆角 24rpx（卡）、9999rpx（按钮、徽章）
- 阴影 8rpx 24rpx 柔焦粉
- 所有 emoji 维持 v1 风格

---

## 12. 异常与边界

| 场景 | UI 处理 |
|------|---------|
| 选加餐时已超所有正餐截止 | 不显示延期提示（加餐无阈值）|
| 周末访问 | 顶部提示 + 周末打卡不纳入 |
| 节假日（未识别） | 视为工作日，参与统计 |
| 跨周打开 | 上一周数据保留，标"上周" |
| 配对码状态 | 与 v1 一致 |
