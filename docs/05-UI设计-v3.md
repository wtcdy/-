# UI 设计 v3.1 · 重设计方案

> 在 v3 基础上做 **3 项收口**：
> 1. 拍照页三类标签全部"必选"（餐次/正餐速食/食物类别）
> 2. 历史记录页去除旧细粒度食物标签（米饭菜蛋奶），**仅保留 5 大类别 + 按时/延时**
> 3. 修复 phone 元素 `overflow: hidden` 导致下半屏被裁切的问题（拍照页必须能完整看到 5 大食物类别）
>
> v3 增量：在 v2 基础上把"食物类型"区从 10 个细粒度改为 5 大类别多选（主食/蔬菜水果/肉蛋水产/汤粥饮品/甜品零食）。其他 11 屏设计**保持 v2 不变**。

## 1. 视觉令牌扩展

### 1.1 新增状态色

| 状态 | 主色 | 浅色背景 | 含义 |
|------|------|----------|------|
| 按时 `on_time` | `#5C8C72` 薄荷绿 | `rgba(200,230,217,0.3)` | 截止时间内 |
| 延期 `late` | `#B58A3F` 蜜桃橙 | `rgba(255,217,179,0.3)` | 超时 1h 内 |

### 1.2 主分类色

| 分类 | 背景 | 文字 |
|------|------|------|
| 正餐 `meal` | `#FFE3E8` | `#FF6B8B` |
| 速食 `fast` | `#FFD9B3` | `#B58A3F` |

### 1.3 食物类别色【v3 新增】

| 类别 | emoji | 渐变 | 主色 |
|------|-------|------|------|
| 主食 `staple` | 🍚 | `#FFF4D6 → #FFE7A0` | `#F0C95C` 奶油黄 |
| 蔬菜水果 `veggie` | 🥗 | `#D6EBDC → #C6EBC9` | `#7BC094` 薄荷绿 |
| 肉蛋水产 `protein` | 🍗 | `#FFE0CC → #FFC7A0` | `#F0A36B` 蜜桃橙 |
| 汤粥饮品 `soup` | 🍵 | `#DCE8F4 → #C7E0F0` | `#7AAED1` 雾霾蓝 |

> **设计原则**：5 个色相均匀分布在马卡龙色环上，避免撞色；每类一个 emoji 锚点，便于盲扫。

## 9. 关键交互流

### 9.1 拍照 → 阈值反馈

```
用户打开拍照页
  → 选餐次
  → 实时计算状态
  → 选正餐/速食（必选）
  → 选食物类型
  → 写备注
  → 看到"当前 14:30，午餐 14:00 截止，已延期 30min"提示
  → 点击保存
  → 服务器再次校验状态（防作弊）
  → 写库
  → 返回首页，看到打卡卡边框为蜜桃橙 + 🟡 延期 30min
```

### 9.2 首页 → 周总结

```
用户点"本周小结卡"
  → 进入周总结页
  → 加载 getWeeklySummary
  → 显示大圆环 + 分布 + Top3 + 5 日日历
  → 点"查看历史" → 跳历史页
```

### 9.3 周末访问

```
周六打开小程序
  → 首页本周小结卡显示"周末好好休息～"
  → 周总结页顶部提示"今天是周末，工作日统计明天更新"
  → 周末打卡仍可记录，但只展示在历史中，不纳入统计
```

## 11. 视觉风格一致

- 维持梦幻治愈 + 马卡龙色
- 状态色薄荷绿/蜜桃橙/雾霾蓝/浅灰 都是已有色板
- 圆角 24rpx（卡）、9999rpx（按钮、徽章）
- 阴影 8rpx 24rpx 柔焦粉
- 所有 emoji 维持 v1 风格

## 12. 异常与边界

| 场景 | UI 处理 |
|------|---------|
| 选加餐时已超所有正餐截止 | 不显示延期提示（加餐无阈值）|
| 周末访问 | 顶部提示 + 周末打卡不纳入 |
| 节假日（未识别） | 视为工作日，参与统计 |
| 跨周打开 | 上一周数据保留，标"上周" |
| 配对码状态 | 与 v1 一致 |
