# 最佳干饭人 · 部署指南

> 从代码到上线的完整步骤，按顺序操作。

---

## 📋 部署流程总览

```
① 申请云开发环境
    ↓
② 修改 config/app.js 切换到 cloud 模式
    ↓
③ 部署 12 个云函数
    ↓
④ 初始化 6 个数据库集合 + 索引
    ↓
⑤ 配置云存储
    ↓
⑥ 开通内容安全接口
    ↓
⑦ 上传协议文档到云存储
    ↓
⑧ IDE 中点击"上传"→ 提交体验版
    ↓
⑨ mp.weixin.qq.com 提交审核
    ↓
⑩ 审核通过 → 发布
```

预计总耗时：第 1 次 **1-2 小时**，熟练后 **30 分钟**。

## ① 申请云开发环境

### 步骤 1：进入云开发控制台

1. 打开**微信开发者工具**
2. 顶部工具栏点击"**云开发**"按钮
3. 弹出"开通云开发"对话框

### 步骤 2：选择计费方式

- **个人主体建议**：选择"**按量付费**"
  - 免费额度：数据库 2GB、存储 5GB、云函数 4 万次/月
  - MVP 阶段完全免费
  - 后续可升级为"预付费"享受折扣

### 步骤 3：创建环境

| 字段 | 填写 |
|---|---|
| 环境名称 | `ganfan-prod`（或 `ganfan-dev`） |
| 环境付费 | 按量付费 |
| 环境版本 | 基础版 |

### 步骤 4：记录环境 ID

创建完成后，控制台"**概览**"页右上角会显示环境 ID：

```
env-id: ganfan-prod-xxxxxx
         ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑
         这就是您的环境 ID
```

> ⚠️ **请记下来**，后面要用。

## ② 修改 config/app.js

打开项目根目录的 `config/app.js`，修改两处：

```javascript
// 第 6 行
mode: 'cloud',   // 改为 'cloud'

// 第 11 行
cloudEnv: 'ganfan-prod-xxxxxx',   // 改为您刚申请的环境 ID
```

保存后，`app.js` 启动时就会调用 `wx.cloud.init()`。

## ③ 部署 12 个云函数

### 为什么需要"上传并部署：云端安装依赖"

每个云函数目录下都有 `package.json`：
```json
{
  "name": "login",
  "version": "1.0.0",
  "dependencies": {
    "wx-server-sdk": "~2.6.3"
  }
}
```

"**云端安装依赖**" 会自动在云端 `npm install` 这个 SDK。

### 步骤 1：右击云函数

在 IDE 左侧文件树中：
- 找到 `cloudfunctions/login/`
- **右键** → "**上传并部署：云端安装依赖**"

### 步骤 2：等待上传

- 首次上传会慢一些（30-60 秒）
- 状态显示"上传中" → "部署成功"

### 步骤 3：重复 12 次

依次部署以下 12 个云函数：

| 顺序 | 云函数 | 顺序 | 云函数 |
|---|---|---|---|
| 1 | `login` | 7 | `pairUnbind` |
| 2 | `createRecord` | 8 | `getPartner` |
| 3 | `getRecords` | 9 | `getRecordDetail` |
| 4 | `getTodayRecords` | 10 | `getCurrentWeekStats` |
| 5 | `pairCreate` | 11 | `getWeeklySummary` |
| 6 | `pairJoin` | 12 | `sendLike` |

### 步骤 4：验证

云开发控制台 → "**云函数**" 页 → 应看到 12 个函数状态都是"**正常**"。

### 常见问题

| 报错 | 解决 |
|---|---|
| `Cannot find module 'wx-server-sdk'` | 上传时未选"云端安装依赖"，重新上传 |
| `permission denied` | 检查 IDE 是否已登录正确 AppID |
| `函数已存在` | 这是正常的，覆盖即可 |

## ④ 初始化 6 个数据库集合

### 步骤 1：进入数据库

云开发控制台 → "**数据库**" → "**+ 添加集合**"

### 步骤 2：创建集合

依次创建 6 个集合（**字段无需预先建**，代码首次写入时自动建）：

| 集合名 | 用途 |
|---|---|
| `users` | 用户档案 |
| `pairCodes` | 配对码（6 位） |
| `meals` | 打卡记录 |
| `likedPartnerDates` | 点赞记录 |
| `dailyStats` | 日统计（预留） |
| `weeklyStats` | 周统计（预留） |

### 步骤 3：设置权限

**统一设置**为"**仅创建者可读写**"（最严格）。如果之后有需要再调整。

### 步骤 4：创建索引（重要！）

为常用查询字段创建索引，提升性能：

| 集合 | 索引字段 | 类型 | 说明 |
|---|---|---|---|
| `users` | `openid` | 唯一 | 用户唯一标识 |
| `users` | `partnerOpenid` | 普通 | 反向查搭子 |
| `pairCodes` | `code` | 唯一 | 配对码唯一 |
| `pairCodes` | `expireAt` | 普通 | TTL 过期清理 |
| `meals` | `openid` + `date` | 复合 | 个人某天记录 |
| `meals` | `targetOpenid` + `date` | 复合 | 搭子某天记录 |
| `likedPartnerDates` | `openid` + `date` | 唯一 | 每天限 1 次 |

**操作方法**：
1. 选中集合 → "**索引管理**" → "**+ 添加索引**"
2. 填写字段名和类型
3. 复合索引：填两个字段名，逗号分隔

### 步骤 5：TTL 索引（配对码自动清理）

`pairCodes` 集合的 `expireAt` 字段需要设置**TTL 索引**：
- 索引类型：选 "**TTL**"
- 过期时间字段：`expireAt`
- 过期后 N 秒删除：填 `0`（过期立即删）

## ⑤ 配置云存储

### 步骤 1：进入云存储

云开发控制台 → "**存储**"

### 步骤 2：创建目录

在根目录创建：
```
meal-images/    ← 打卡图片
agreements/     ← 协议文档
```

### 步骤 3：设置权限

| 目录 | 权限 | 说明 |
|---|---|---|
| `meal-images/` | 仅创建者可读写 | 用户只能读自己上传的 |
| `agreements/` | 所有用户可读 | 协议文档需公开可访问 |

### 步骤 4：上传协议文档

把以下 2 个文件上传到 `agreements/` 目录：

```
agreements/user-agreement.md
agreements/privacy-policy.md
```

> 这样小程序里可以通过云存储 URL 访问，避免代码里硬编码长文本。

## ⑥ 开通内容安全接口

> **重要！** 个人主体也必须做，否则审核可能被拒。

### 步骤 1：进入设置

云开发控制台 → "**设置**" → "**拓展功能**"

### 步骤 2：开通

找到"**内容安全**"，点击"**开通**"。

| 接口 | 用途 | 调用方 |
|---|---|---|
| `imgSecCheck` | 检测图片违规 | `createRecord` 云函数 |
| `textSecCheck` | 检测文字违规 | `createRecord` 云函数 |
| `msgSecCheck` | 检测一段文本（更长） | 留言功能（v2 预留） |

### 步骤 3：验证

开通后，云函数内调用 `cloud.openapi.security.imgSecCheck` 应该能成功。

## ⑦ 上传体验版

### 步骤 1：检查项目配置

打开 `project.config.json`，确认：
- `appid`: `wx15e191c1fb1ab4cc`（您的真实 AppID）
- `miniprogramRoot`: `./`
- `cloudfunctionRoot`: `cloudfunctions/`

### 步骤 2：IDE 中点击"上传"

1. 微信开发者工具顶部 → "**上传**" 按钮
2. 弹窗填写：

| 字段 | 填写 |
|---|---|
| 版本号 | `1.0.0` |
| 项目备注 | `首版上线，含打卡/配对/周统计` |

3. 点击"**上传**"

### 步骤 3：等待上传完成

IDE 控制台会显示上传进度，成功后会显示：
```
已上传到 1.0.0, 用时 8s
```

## ⑧ 提交审核

### 步骤 1：进入 mp 后台

打开 https://mp.weixin.qq.com → 登录 → "**版本管理**"

### 步骤 2：选择体验版

在"**开发版本**"列表中，找到刚才上传的 `1.0.0` 版本，点击"**提交审核**"。

### 步骤 3：填写审核信息

参考 `docs/agreements/store-listing.md` 的内容填写：

| 字段 | 填写 |
|---|---|
| 功能页面 | 选择主要页面截图 |
| 类目 | 工具 → 健康管理 |
| 标签 | 健康、工具、记录、生活 |
| 测试账号 | （个人主体可能不需要） |
| 审核备注 | 复制 `store-listing.md` 里的"审核备注"段 |

### 步骤 4：提交

点击"**提交审核**"按钮，等待审核结果（一般 1-3 个工作日）。

---

## ⑨ 审核结果处理

### 通过 ✅

mp 后台 → "**版本管理**" → "**审核版本**" → "**发布**"

### 被拒 ❌

常见原因及对策（详见 `docs/agreements/store-listing.md`）：

| 被拒原因 | 对策 |
|---|---|
| 类目不符 | 修改类目描述，强调"个人记录工具" |
| 涉及社交 | 强调"1:1 私密配对，非公开社区" |
| 缺少协议 | 重新提交，确保协议 URL 可访问 |
| 强制授权 | 提供"拒绝"入口 |

## 🆘 部署过程问题排查

### Q1：云函数调用报"cloud has not been initialized"

**原因**：`app.js` 中 `mode: 'cloud'` 但 `cloudEnv` 没填。

**解决**：检查 `config/app.js` 中 `cloudEnv` 是不是您的真实环境 ID。

### Q2：调用云函数报"permission denied"

**原因**：云函数部署了，但权限未授权。

**解决**：在 `app.js` 中确认 `wx.cloud.init` 已调用。

### Q3：图片上传失败

**原因**：云存储权限未设置。

**解决**：控制台 → 存储 → 权限设置 → 选"仅创建者可读写"。

### Q4：审核被拒"类目不符"

**原因**：描述里有社交、社区等敏感词。

**解决**：检查 `app.json` 中 `permission` 配置；检查代码注释里有没有"社交"字样。
